# Usage guide

How to set up, run, and use Evollis.

## 1. Prerequisites

- **Node.js 20+** and npm
- **PostgreSQL** database (local or hosted) — you need a connection URL
- An **OpenAI API key** (required: replies are generated by OpenAI)
- *(Optional)* **Python 3.9+** — only if you want to run the BERTopic classifier

## 2. Configuration

Copy the example env file and fill it in:

```bash
cp .env.example .env
```

| Variable | Required | Default | Purpose |
|---|---|---|---|
| `DATABASE_URL` | ✅ | — | PostgreSQL connection string |
| `OPENAI_API_KEY` | ✅ | — | Generates replies; also classifies when provider is `openai` |
| `OPENAI_MODEL` | — | `gpt-5.4-mini-2026-03-17` | Vision-capable model used for classification + replies |
| `CLASSIFIER_PROVIDER` | — | `openai` | `openai` or `bertopic` — which classifier assigns the category |
| `BERTOPIC_SERVICE_URL` | — | `http://127.0.0.1:8000` | Where the BERTopic service listens (only used when provider is `bertopic`) |

> The app validates env vars on boot — it won't start without `DATABASE_URL`
> and `OPENAI_API_KEY`.

## 3. Install and run

```bash
npm install
npm run db:push     # create the tables from prisma/schema.prisma
npm run db:seed     # load demo agents, clients and conversations
npm run dev         # start the app
```

Open **http://localhost:3000**. The app has two surfaces:

- `/` — the **customer** chat
- `/inbox` — the **agent** inbox

Both show a login modal **pre-filled with valid demo credentials**. There is no
password (identity is a demo cookie); just click **Se connecter**.

## 4. Using the customer side (`/`)

1. Sign in (pick a seeded client in the modal, or keep the pre-filled one).
2. Type a message and send it. Your message appears instantly, followed by a
   **"writing…"** indicator.
3. Behind the scenes the whole conversation is **categorised**
   (Billing / Technical / Other) and an answer is generated with prompts
   tailored to that category. The reply then appears in the thread.
4. Use the feedback bar under a reply:
   - **Résolu** — closes the conversation.
   - **Pas résolu** — escalates: the bot stops replying and a human agent is
     handed the thread.
5. Past conversations are listed in the left sidebar.

Every follow-up message re-sends the **full prior conversation** as context, so
the assistant stays coherent across turns.

## 5. Using the agent side (`/inbox`)

1. Go to `/inbox` and sign in as a seeded agent.
2. The list shows conversations with channel, **category chip**
   (*Facturation* / *Technique* / *Général*), priority and unread badges.
3. Filter by tab (mine / unassigned / all) and by status, or search.
4. Open a conversation to read the full history (including AI replies and
   private notes). From the right-hand panel you can:
   - reply publicly or add a private note,
   - change **priority**, **status**, and **assignee**,
   - add/remove **labels** (Libellés),
   - view contact details and previous conversations.

## 6. Switching the classifier to BERTopic (optional)

By default categories are assigned by an OpenAI call. To use the free, local
semantic classifier instead:

```bash
cd ml/bertopic
python -m venv .venv
# Windows:        .venv\Scripts\activate
# macOS / Linux:  source .venv/bin/activate
pip install -r requirements.txt
uvicorn server:app --host 127.0.0.1 --port 8000
```

Then in `.env` set:

```
CLASSIFIER_PROVIDER="bertopic"
BERTOPIC_SERVICE_URL="http://127.0.0.1:8000"
```

Restart `npm run dev`. Replies are still generated by OpenAI; only the category
assignment changes. See [../ml/bertopic/README.md](../ml/bertopic/README.md) for
details and the proprietary-data caveat. If the service is unreachable, the app
logs the error and falls back to an `UNKNOWN` category — a send never fails.

## 7. Useful scripts

| Command | What it does |
|---|---|
| `npm run dev` | Start the dev server |
| `npm run build` / `npm run start` | Production build / serve |
| `npm run db:push` | Apply the schema to the database |
| `npm run db:seed` | Reseed demo data |
| `npm run db:studio` | Open Prisma Studio to inspect the data |
| `npm run typecheck` | TypeScript check |
| `npm run lint` | ESLint |

## 8. Troubleshooting

- **App won't start / "Invalid environment variables"** — `DATABASE_URL` or
  `OPENAI_API_KEY` is missing in `.env`.
- **Categories all come back "Autre"** — classification is failing. For
  `openai`, check the API key/quota; for `bertopic`, check the Python service is
  running at `BERTOPIC_SERVICE_URL`. (`Autre` = `UNKNOWN`, the fallback.)
- **No conversations / empty inbox** — run `npm run db:seed`.
